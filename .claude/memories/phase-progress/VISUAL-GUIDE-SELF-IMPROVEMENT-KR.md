# 자기개선 시스템 시각화 가이드

**버전**: 3.0.0
**작성일**: 2025-10-13
**대상**: 비프로그래머, 일반 사용자

---

## 📖 이 문서는 무엇인가요?

수학 교육 AI 시스템이 **스스로 문제를 발견하고 개선하는 능력**을 갖추게 됩니다.
마치 사람이 실수를 통해 배우는 것처럼, AI도 자신의 성능을 모니터링하고 자동으로 발전합니다.

**핵심 아이디어**: "AI가 AI를 개선한다"

---

## 🎯 왜 필요한가요?

### 기존 시스템 (v2.0)

```
사용자 → AI 시스템 → 답변

문제 발생 시:
❌ 개발자가 직접 코드를 수정해야 함
❌ 문제 원인 파악에 시간 소요
❌ 수동 개입 필요
```

### 새로운 시스템 (v3.0)

```
사용자 → AI 시스템 → 답변
         ↓ (문제 감지)
    자동 분석 시스템
         ↓
    자동 수정 적용
         ↓
    개선된 AI 시스템 → 더 나은 답변

✅ 자동으로 문제 발견
✅ 자동으로 원인 분석
✅ 자동으로 해결 방안 적용
```

## 🔄 작동 흐름 - 시각화

### 정상 작동 (Normal Flow)

```
[1단계] 사용자 질문
   ↓
[2단계] 메타 오케스트레이터가 6개 에이전트에게 작업 분배
   ↓
[3단계] 각 에이전트가 병렬로 작업 수행
   ┌─────────┬─────────┬─────────┬─────────┬─────────┬─────────┐
   │ 지식구축 │ 품질검증 │ 연구조사 │ 예제생성 │ 관계매핑 │ 계획수립 │
   └─────────┴─────────┴─────────┴─────────┴─────────┴─────────┘
   ↓
[4단계] 결과 통합 및 사용자에게 답변 제공
   ↓
[5단계] 성능 지표 기록 (성공률, 응답 시간, 오류 횟수 등)
```

### 문제 발생 시 (Improvement Flow)

```
[감지] 성능 모니터가 문제 발견!
   • 성공률 80% 미만
   • 3회 연속 실패
   • 응답 시간 1.5배 증가

   ↓

[분석 시작] 소크라테스-조정자 에이전트 활성화

   "왜 실패했나요?"        →  문제 에이전트에게 질문
   "이 경우를 고려했나요?"  →  추가 질문
   "원인이 무엇인가요?"     →  근본 원인 파악

   ↓

[원인 보고서 생성]
   ┌────────────────────────────────────┐
   │ 근본 원인 분석 보고서              │
   │ ────────────────────────────────  │
   │ 문제 에이전트: 연구조사            │
   │ 원인: 특정 수학 용어 처리 누락     │
   │ 신뢰도: 85%                        │
   │ 권장사항:                          │
   │  1. 프롬프트에 용어 정의 추가      │
   │  2. 예외 처리 로직 보강            │
   └────────────────────────────────────┘

   ↓

[개선 적용] 자기개선 에이전트 활성화

   • 에이전트 설정 파일 읽기
   • 문제 부분 찾기
   • 수정 사항 적용
   • 변경 이력 저장

   예시:
   기존: "수학 개념을 설명하세요"
    ↓
   개선: "수학 개념을 설명하세요.
         전문 용어는 반드시 정의와 함께 제공하세요."

   ↓

[검증] 개선 효과 테스트
   • 샘플 질문으로 테스트 실행
   • 성능 비교 (개선 전 vs 개선 후)

   ↓
  성공? ──┬── YES → 변경 사항 유지, 정상 플로우 재개
          │
          └── NO  → 자동 롤백 (이전 상태로 복구)
```

## 👥 새로운 AI 에이전트 소개

### 7번 에이전트: 소크라테스-조정자 (Socratic-Mediator)

```
역할: 철학자처럼 질문하며 문제의 근본 원인을 찾는 탐정

특징:
• 직접 답을 주지 않고 질문으로 유도
• 다른 에이전트들과 대화하며 논리 점검
• 최대 10개 질문까지 허용
• 최종 원인 분석 보고서 작성

도구:
✓ ask_agent - 다른 에이전트에게 질문하기
✗ 파일 수정 권한 없음 (분석만 담당)

예시 대화:
소크라테스: "연구조사 에이전트님, 왜 이 용어를 인식하지 못했나요?"
연구조사: "프롬프트에 해당 용어 처리 지침이 없습니다."
소크라테스: "그럼 어떤 지침이 있었다면 해결되었을까요?"
연구조사: "용어 정의를 먼저 검색하라는 지침이요."
소크라테스: "알겠습니다. 근본 원인은 프롬프트 누락이군요."
```

### 8번 에이전트: 자기개선 (Self-Improver)

```
역할: 시스템 정비사, 문제를 실제로 고치는 엔지니어

특징:
• 소크라테스의 분석 보고서를 받아 실행
• 에이전트 설정 파일 직접 수정
• 필요 시 새로운 에이전트 생성
• 모든 변경 사항 기록 및 추적

도구:
✓ Read - 파일 읽기
✓ Write - 새 파일 생성
✓ Edit - 기존 파일 수정
✓ Grep - 코드 검색
✓ Glob - 파일 찾기

안전 장치:
• 최대 세션당 5회 개선만 허용
• 변경 전 반드시 백업
• 검증 실패 시 자동 롤백
• 모든 변경 이력 영구 저장
```

## 🛡️ 안전 장치

시스템이 스스로를 망가뜨리지 않도록 여러 안전 장치가 있습니다.

### 1. 개선 횟수 제한

```
세션당 최대 5회까지만 개선 가능

이유: 무한 루프 방지
     "개선 → 실패 → 개선 → 실패..." 반복 차단

[개선 1] ✅
[개선 2] ✅
[개선 3] ✅
[개선 4] ✅
[개선 5] ✅
[개선 6] ❌ 차단! "오늘은 여기까지만 개선하겠습니다"
```

### 2. 롤백 시스템

```
변경 전 상태를 항상 저장

개선 적용 → 문제 발생? → 자동 롤백

예시:
┌─────────────────────────────────────┐
│ 변경 이력 #47                       │
│ 시간: 2025-10-13 14:30:00           │
│ 대상: 연구조사 에이전트             │
│ 변경 전: "기본 설명 제공"           │
│ 변경 후: "상세 설명 + 용어 정의"    │
│ 상태: 적용됨 ✅                     │
└─────────────────────────────────────┘
        ↓ (문제 발생 시)
┌─────────────────────────────────────┐
│ 롤백 실행!                          │
│ "변경 후" → "변경 전"으로 복구      │
│ 상태: 롤백됨 ↩                      │
└─────────────────────────────────────┘
```

### 3. 검증 테스트

```
모든 개선 후 반드시 테스트

[개선 적용]
   ↓
[테스트 질문 실행] "피타고라스 정리란?"
   ↓
[결과 확인]
   • 답변 생성됨? ✓
   • 응답 시간 정상? ✓
   • 품질 기준 충족? ✓
   ↓
[검증 통과] → 개선 유지

만약 하나라도 실패 → 자동 롤백
```

### 4. 신뢰도 점수

```
소크라테스 분석 시 신뢰도 점수 부여

신뢰도 85% → "매우 확실함" → 개선 적용 ✅
신뢰도 60% → "불확실함"   → 사용자에게 확인 요청 ⚠️
신뢰도 40% → "추측일 뿐"  → 개선 거부 ❌

낮은 신뢰도로 함부로 변경하지 않음!
```

## ⚙️ 설정 가능한 항목

사용자나 관리자가 조정할 수 있는 설정들:

### 1. 개선 트리거 조건

```python
improvement_thresholds = {
    "min_success_rate": 0.80,      # 80% 미만 시 트리거
    "max_consecutive_failures": 3,  # 3회 연속 실패 시
    "performance_regression": 1.5,  # 1.5배 느려지면
}

조정 가능:
- 더 민감하게: min_success_rate = 0.90 (90%)
- 더 관대하게: min_success_rate = 0.70 (70%)
```

### 2. 자동 실행 여부

```python
auto_improvement = False  # 기본값: 사용자 확인 요청

True로 설정 시:
  문제 감지 → 즉시 자동 개선 (확인 안 함)

False로 설정 시:
  문제 감지 → 사용자에게 확인 요청
```

### 3. 개선 횟수 제한

```python
max_improvements_per_session = 5  # 세션당 5회

조정 가능:
- 더 많이: 10회
- 더 적게: 3회
- 무제한: 999회 (권장하지 않음)
```
